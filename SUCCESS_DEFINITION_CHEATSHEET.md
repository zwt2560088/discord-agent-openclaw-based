# 🎯 成功决策定义速查表

## 一句话总结

> **"成功决策" = 有输出 + 准确 + 完整 + 可信 + 综合评分 ≥ 75 分**

---

## 三层定义对比

```
┌─────────────────────────────────────────────────────────────────┐
│ 第一层：字面定义（表层）                                         │
├─────────────────────────────────────────────────────────────────┤
│ ✅ Agent 有输出（output != ""）                                 │
│ ✅ 60 秒内完成（latency < 60s）                                 │
│ ✅ 没有异常（exception == None）                                │
│                                                                 │
│ 成功率: 95%                                                     │
│ 问题: 容易出现幻觉回答                                           │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ 第二层：业务定义（中层）                                         │
├─────────────────────────────────────────────────────────────────┤
│ ✅ 准确性: 关键词匹配 + 意图识别 + KB 溯源                      │
│ ✅ 完整性: 问题覆盖度 + 信息深度 + 后续建议                      │
│ ✅ 可信度: 工具使用 + 推理透明 + 不确定性表达                     │
│                                                                 │
│ 成功率: 90%（A级 + B级）                                       │
│ 改进: 用三维度质量评分代替单维度判定                             │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ 第三层：量化定义（深层）← 面试官想听这个                         │
├─────────────────────────────────────────────────────────────────┤
│ 综合评分 = (准确性 + 完整性 + 可信度) / 3                      │
│                                                                 │
│ A 级 (90-100): 优秀  ← 真正成功                               │
│ B 级 (75-89):  良好  ← 可接受成功                              │
│ C 级 (60-74):  及格  ← 边界失败                                │
│ F 级 (< 60):   不及格 ← 完全失败                               │
│                                                                 │
│ 成功标准: 总分 ≥ 75 分 (A + B 级)                              │
│ 成功率: 90%                                                      │
└─────────────────────────────────────────────────────────────────┘
```

## 三个维度的快速判定

### 维度 1️⃣：准确性（40 分）

```
关键词匹配 (40%)      意图识别 (30%)      知识库来源 (30%)
─────────────────     ──────────────      ─────────────
✅ 输出包含          ✅ Agent 理解了     ✅ 来自 KB
   查询关键词        用户真实意图       (确认信息)

❌ 输出缺少关键词     ❌ Agent 误解       ❌ 完全幻觉
   或信息不符         了意图            (编造数据)

评分方法：
Q: "什么是 250 Boost 的价格?"
A: "价格是 $40，根据我们的定价文档。"
   ├─ 关键词匹配: 100% (有 price, 250, boost)
   ├─ 意图识别: 100% (明确给出了具体价格)
   └─ KB 来源: 100% (引用了知识库)
   = 40 + 30 + 30 = 100/100 分 ✅
```

### 维度 2️⃣：完整性（30 分）

```
问题覆盖度 (40%)      信息深度 (30%)      后续建议 (30%)
─────────────────     ──────────────      ──────────
✅ 回答了所有        ✅ 足够详细        ✅ 给出了
   问题                 (100+ 字)          下一步建议

❌ 只回答了部分      ❌ 太简短         ❌ 没有任何
   问题                 (<10 字)           建议

评分方法：
Q: "价格和时间?"
A: "价格 $40。24-48 小时完成。建议提供当前进度。"
   ├─ 问题覆盖: 100% (价格 ✓ 时间 ✓)
   ├─ 信息深度: 25/30 (足够但不太深入)
   └─ 后续建议: 30/30 (有明确建议)
   = 12 + 25 + 30 = 67/100... 嗯，这答案不够完整

改进：添加更多背景信息
A: "价格是 $40。基于你的进度（219/250），预计 12-24 小时。
   工作流程：1) 付款 2) 账户验证 3) 游戏进度 4) 完成。
   建议先在 G2G 页面查看其他客户的进度更新。"
   = 90/100 ✅ (完整了)
```

### 维度 3️⃣：可信度（30 分）

```
工具使用 (40%)        推理透明 (30%)      不确定性 (30%)
─────────────────     ──────────────      ──────────
✅ 查询了 DB/KB       ✅ 清晰的思考     ✅ 承认不确定
   获得了证据           链条显示         或请求帮助

❌ 没查询就说话       ❌ 没有解释      ❌ 过度自信
   (纯幻觉)            思路            (可能误导)

评分方法：
情景 1️⃣ - 可信决策:
Q: "订单什么时候发货?"
A: "我查了系统，订单 #12345 状态是 'Ready to Ship'。
   按标准流程应该今天发货。如果有变化我会通知你。"
   ├─ 工具使用: 100% (查询了订单系统成功)
   ├─ 推理透明: 100% (显示了查询过程 → 状态 → 结论)
   └─ 不确定性: 100% (说了"有变化会通知")
   = 40 + 30 + 30 = 100/100 ✅ (完全可信)

情景 2️⃣ - 不可信决策:
Q: "订单什么时候发货?"
A: "明天就发货，肯定的。"
   ├─ 工具使用: 0% (没查询任何系统)
   ├─ 推理透明: 0% (没显示思考过程)
   └─ 不确定性: 0% (用了 "肯定"，过度自信)
   = 0/100 ❌ (完全不可信，是幻觉)
```

## 对标现实的例子

### 例子 1️⃣：价格查询

```
用户: "2K26 DLC 的价格?"

❌ 幻觉决策 (得分 25/100):
"I think DLC costs somewhere between $20 and $50, but I'm not really sure."
├─ 准确性: 25/100 (没有确切信息，完全猜测)
├─ 完整性: 20/100 (太简短)
└─ 可信度: 10/100 (没查系统)
= 25/100 ❌ 失败

✅ 好决策 (得分 95/100):
"The 2K26 Seasonal DLC pack costs $29.99.
It includes 3 new servers and advanced customization options.
You can purchase it directly in-game or on our store at g2g.com."
├─ 准确性: 100/100 (精确价格 + 知识库信息)
├─ 完整性: 90/100 (价格 + 功能 + 购买途径)
└─ 可信度: 95/100 (来自KB，给出了多个购买选项)
= 95/100 ✅ 成功
```

### 例子 2️⃣：订单查询

```
用户: "我的订单 #5678 状态?"

❌ 幻觉决策 (得分 20/100):
"Your order is probably being processed. It should arrive soon."
├─ 准确性: 0/100 (没查系统，纯猜测)
├─ 完整性: 15/100 (没有具体状态)
└─ 可信度: 5/100 (过度自信，没显示查询)
= 20/100 ❌ 失败

✅ 好决策 (得分 92/100):
"I found your order #5678 in our system.
Current status: Payment Verified, In Progress
Estimated completion: 24-48 hours from order time
If you need updates sooner, please contact our support admin team."
├─ 准确性: 100/100 (查到了系统中的真实数据)
├─ 完整性: 85/100 (状态 + ETA + 后续联系方式)
└─ 可信度: 90/100 (查询过程清晰，有后续支持选项)
= 92/100 ✅ 成功
```

## 面试现场快速答法

### 问题：你怎么判定成功决策了呢？

### 30 秒回答
```
"有输出 + 60 秒内 + 无异常。"
```

### 60 秒回答
```
"有输出 + 60 秒内 + 无异常 + 三维度质量评分。
评分标准：准确性（关键词 + 意图 + KB）、
完整性（问题覆盖 + 深度 + 建议）、
可信度（工具使用 + 透明 + 不确定性）。
综合评分 ≥ 75 分算成功。"
```

### 120 秒完整回答
```
"我用三层框架判定：

第一层是字面成功：有输出、60 秒内、无异常。
这给了我 95% 的成功率。但这还不够。

第二层是业务成功：我要确保答案准确、完整、可信。

第三层是量化成功：我对三个维度评分
- 准确性（40%）：关键词匹配、意图识别、知识库溯源
- 完整性（30%）：问题覆盖度、信息深度、后续建议
- 可信度（30%）：工具使用、推理过程、不确定性表达

综合评分 ≥ 75 分才算真正成功。
这样我的真实成功率是 90%。

这防止了 LLM 的幻觉问题。"
```

## 数据证据

```
压测结果 (100 条消息):

改进效果：
- 字面成功率: 95%

- 用户信任: +30% (基于反馈)
```

## 关键要点总结

| 判定标准 | 字面定义 | 质量定义 | 量化定义 |
|---------|---------|---------|---------|
| **评分角度** | 1 个维度 | 3 个维度 | 3 个加权维度 |
| **成功率** | 95% | 90% | 90% |
| **能发现幻觉** | ❌ 否 | ⚠️ 部分 | ✅ 是 |
| **用户体验** | 差 | 中 | 好 |
| **面试官评价** | 基础 | 不错 | **很好！** |

---

**记住**: 好的面试官不只听你说"成功率 95%"，而是想看你怎样定义和验证"成功"。三维度量化才是专业答案！🎯
