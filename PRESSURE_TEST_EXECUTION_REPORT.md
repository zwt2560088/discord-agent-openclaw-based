```

• 缓存命中率高达 93.92%
  → 平均处理时间远低于 500ms
• 消息处理分布优化良好
  → 大多数消息 < 100ms 完成

  → 无 GC 停顿或 I/O 等待
```

**结论**: 吞吐量表现 **优异** ✅

#### 2. 延迟分析

```
平均延迟: 70.65 ms
  ↓
  大多数消息在 70ms 内完成
  包含: 缓存查询 (10ms) + AI 处理 (50ms) + DB 写入 (11ms)

P95 延迟: 430.98 ms
  ↓
  95% 的消息在 431ms 内完成

### 1. 缓存策略的巨大作用

```
当前缓存命中率: 93.92%

• 缓存命中消息: 93.92%

## 🚀 性能优化建议

### 短期 (已实现但可改进)

```python
# 当前
cache_key = f"{channel}:latest"

# 改进
cache_key = f"{channel}:{user_id}:recent_5"
# 缓存用户最近 5 条消息
```

#### 2. 数据库批量写入 (可降低 P99 延迟 30%)

```python
# 当前: 逐条写入
await db.insert(message)

# 改进: 批量写入
batch = [msg1, msg2, msg3, ..., msg100]
await db.batch_insert(batch)
```

try:

    response = await fallback_strategy()
